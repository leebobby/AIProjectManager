## 1. 架构概述

```
                     ┌─────────────────────────────┐
  浏览器 60人  ──────▶│         Nginx               │
                     │  • 静态文件 (Vue 前端)        │
                     │  • 反向代理 /api → 后端       │
                     └────────────┬────────────────┘
                                  │
                     ┌────────────▼────────────────┐
                     │   Uvicorn + FastAPI 后端     │
                     │   监听 127.0.0.1:8000        │
                     └────────────┬────────────────┘
                                  │
                     ┌────────────▼────────────────┐
                     │   SQLite  (app.db)           │
                     │   config.json               │
                     └─────────────────────────────┘
```

**技术栈**
- 后端：Python 3.11 + FastAPI + Uvicorn
- 前端：Vue 3 + Vite（构建为静态文件，由 Nginx 直接服务）
- 数据库：SQLite（文件型，零维护）
- 反向代理：Nginx

## 3. 首次部署

> 以下所有命令在 **服务器** 上以 `ubuntu`（或其他普通用户）执行，需要 root 权限的步骤会显式使用 `sudo`。

### 3.1 安装系统依赖

```bash
sudo apt update && sudo apt upgrade -y

# Python 3.11
sudo apt install -y python3.11 python3.11-venv python3-pip

# Node.js 18 LTS（通过 NodeSource）
curl -fsSL https://deb.nodesource.com/setup_18.x | sudo -E bash -
sudo apt install -y nodejs

# Nginx & 工具
sudo apt install -y nginx git sqlite3
```

验证安装：

```bash
python3.11 --version   # Python 3.11.x
node -v                 # v18.x.x
nginx -v                # nginx/1.x.x
```

### 3.2 拉取代码

```bash
# 选择一个合适的目录，例如 /opt
sudo mkdir -p /opt/aipm
sudo chown $USER:$USER /opt/aipm

cd /opt/aipm
git clone <你的仓库地址> app
# 或者直接上传压缩包后解压：
# unzip aipm.zip -d app
```

目录结构应如下：

```
/opt/aipm/app/
├── backend/          ← Python 后端
│   ├── main.py
│   ├── requirements.txt
│   ├── config.json   ← 会在首次启动后自动创建或手动编辑
│   └── app.db        ← 数据库（首次启动后自动创建）
└── frontend/         ← Vue 前端源码
    ├── src/
    └── package.json
```

### 3.3 部署后端

```bash
cd /opt/aipm/app/backend

# 创建虚拟环境
python3.11 -m venv venv
source venv/bin/activate

# 安装依赖
pip install -r requirements.txt

# 编辑配置文件（关键：修改 issue_report_path 为服务器上的报表目录）
cp config.json.example config.json  # 如有示例文件
# 或者直接编辑：
nano config.json
```

`config.json` 示例：

```json
{
  "current_stages": ["BFI", "岳麓山", "明场"],
  "issue_report_path": "/opt/aipm/reports",
  "about_content": "岳麓山项目管理系统",
  "hero_badges": ["项目名称", "团队名称", "版本", "阶段"]
}
```

**创建 systemd 服务**（让后端开机自启并自动重启）：

```bash
sudo nano /etc/systemd/system/aipm-backend.service
```

填入以下内容：

```ini
[Unit]
Description=岳麓山项目管理系统后端
After=network.target

[Service]
Type=simple
User=ubuntu
WorkingDirectory=/opt/aipm/app/backend
Environment="PATH=/opt/aipm/app/backend/venv/bin"
ExecStart=/opt/aipm/app/backend/venv/bin/uvicorn main:app \
    --host 127.0.0.1 \
    --port 8000 \
    --workers 2
Restart=always
RestartSec=5
StandardOutput=journal
StandardError=journal

[Install]
WantedBy=multi-user.target
```

> `--workers 2`：2 个 worker 进程对 60 人的并发完全够用，内存占用约 200 MB。

```bash
# 启用并启动服务
sudo systemctl daemon-reload
sudo systemctl enable aipm-backend
sudo systemctl start aipm-backend

# 确认运行状态
sudo systemctl status aipm-backend
```

### 3.4 构建前端

```bash
cd /opt/aipm/app/frontend

# 安装依赖
npm install

# 构建生产版本
npm run build
# 构建产物输出到 frontend/dist/
```

### 3.5 配置 Nginx

```bash
sudo nano /etc/nginx/sites-available/aipm
```

填入：

```nginx
server {
    listen 80;
    server_name _;          # 替换为实际 IP 或内网域名，如 192.168.1.100

    # 前端静态文件
    root /opt/aipm/app/frontend/dist;
    index index.html;

    # Vue Router history 模式支持
    location / {
        try_files $uri $uri/ /index.html;
    }

    # 反向代理到后端 API
    location /api/ {
        proxy_pass         http://127.0.0.1:8000;
        proxy_set_header   Host $host;
        proxy_set_header   X-Real-IP $remote_addr;
        proxy_read_timeout 120s;
        client_max_body_size 20M;
    }
}
```

```bash
# 启用站点，禁用默认站点
sudo ln -s /etc/nginx/sites-available/aipm /etc/nginx/sites-enabled/
sudo rm -f /etc/nginx/sites-enabled/default

# 检查配置语法
sudo nginx -t

# 重载 Nginx
sudo systemctl reload nginx
sudo systemctl enable nginx
```

### 3.6 验证部署

在同网段另一台电脑浏览器访问 `http://<服务器IP>`，能看到登录页则部署成功。

默认管理员账号：
- 用户名：`admin`
- 密码：`admin123`

**首次登录后请立即修改默认密码。**

## 4. 数据备份与恢复

### 4.1 需要备份什么

| 文件 | 位置 | 重要性 |
|------|------|--------|
| `app.db` | `backend/app.db` | ★★★ 所有业务数据 |
| `config.json` | `backend/config.json` | ★★☆ 配置参数 |
| 报表文件 | `issue_report_path` 指定目录 | ★★☆ 原始数据 |

### 4.2 SQLite 在线备份

SQLite 提供 `.backup` 命令，支持**数据库运行时**安全备份（不需要停服）：

```bash
sqlite3 /opt/aipm/app/backend/app.db ".backup /opt/aipm/backups/app-$(date +%Y%m%d-%H%M%S).db"
```

### 4.3 自动定时备份（推荐）

创建备份脚本：

```bash
sudo mkdir -p /opt/aipm/backups
sudo chown $USER:$USER /opt/aipm/backups

nano /opt/aipm/backup.sh
```

脚本内容：

```bash
#!/bin/bash
set -e

BACKUP_DIR="/opt/aipm/backups"
DB_PATH="/opt/aipm/app/backend/app.db"
CFG_PATH="/opt/aipm/app/backend/config.json"
DATE=$(date +%Y%m%d-%H%M%S)

# 备份数据库（在线热备份）
sqlite3 "$DB_PATH" ".backup ${BACKUP_DIR}/app-${DATE}.db"

# 备份配置文件
cp "$CFG_PATH" "${BACKUP_DIR}/config-${DATE}.json"

# 保留最近 30 份，删除更旧的
find "$BACKUP_DIR" -name "app-*.db"       -mtime +30 -delete
find "$BACKUP_DIR" -name "config-*.json"  -mtime +30 -delete

echo "[$(date)] 备份完成：app-${DATE}.db"
```

```bash
chmod +x /opt/aipm/backup.sh

# 测试一次
/opt/aipm/backup.sh
```

注册 cron 定时任务（每天凌晨 2:00 执行）：

```bash
crontab -e
```

添加：

```
0 2 * * * /opt/aipm/backup.sh >> /var/log/aipm-backup.log 2>&1
```

### 4.4 从备份恢复

**停服 → 替换数据库文件 → 重启**：

```bash
# 1. 停止后端服务
sudo systemctl stop aipm-backend

# 2. 替换数据库（先保留损坏的文件以备查）
cp /opt/aipm/app/backend/app.db /opt/aipm/app.db.broken
cp /opt/aipm/backups/app-20260517-020001.db /opt/aipm/app/backend/app.db

# 3. 重启服务
sudo systemctl start aipm-backend
```

### 4.5 建议：异地备份

将 `/opt/aipm/backups/` 目录定期同步到另一台机器或 NAS，防止服务器硬盘故障导致数据和备份同时丢失：

```bash
# 使用 rsync 同步到另一台机器（每天凌晨 3:00）
# 在 crontab 中追加：
0 3 * * * rsync -az /opt/aipm/backups/ backup-server:/backup/aipm/
```

## 5. 升级流程

### 5.1 升级前准备

每次升级前务必先备份：

```bash
/opt/aipm/backup.sh
```

### 5.2 标准升级步骤

```bash
cd /opt/aipm/app

# 1. 拉取最新代码
git pull origin main

# 2. 更新后端依赖（若 requirements.txt 有变化）
cd backend
source venv/bin/activate
pip install -r requirements.txt

# 3. 重启后端（自动执行数据库迁移）
sudo systemctl restart aipm-backend
sudo systemctl status aipm-backend   # 确认启动正常

# 4. 重新构建前端
cd ../frontend
npm install           # 依赖有变化时自动处理
npm run build

# 5. 刷新 Nginx（静态文件已更新，无需重启，reload 即可）
sudo systemctl reload nginx
```

> 前端构建产物直接覆盖 `dist/`，Nginx reload 后用户刷新浏览器即可看到新版本。整个升级过程后端停服不超过 10 秒。

### 5.3 数据库结构变更

本系统在 `migrate.py` 中维护了轻量迁移逻辑，**重启后端会自动执行**，无需手动运行迁移脚本。

若某次升级后出现 `500` 错误，优先查看后端日志：

```bash
sudo journalctl -u aipm-backend -n 100 --no-pager
```

### 5.4 特定版本升级说明

部分版本涉及数据格式或行为变更，升级时需额外注意。

---

#### v0.10.x → 客户面状态清单字段格式变更

**变更内容**

`customer_status` 表的两个文本字段存储格式由纯文本改为 JSON 数组：

| 字段 | 旧格式（纯文本） | 新格式（JSON） |
|------|-----------------|----------------|
| `recent_focus` | `"良率提升\n备件采购"` | `[{"text":"良率提升","done":false},{"text":"备件采购","done":false}]` |
| `key_issues` | `"主控板偶发复位"` | `[{"text":"主控板偶发复位","done":false}]` |

**是否需要修改数据库？不需要。**

两个字段仍然是 `TEXT` 类型，表结构无任何变化。旧数据可直接使用，无需运行任何迁移脚本。

**兼容原理**

前端在读取字段时会自动判断格式：

```
读取字段值
    ├── 能解析为 JSON 数组 → 直接作为清单条目渲染（新格式）
    └── 否则按换行符拆分  → 每行作为一个未勾选条目渲染（旧格式自动兼容）
```

旧数据在用户第一次对该行执行操作（勾选、新增条目、点「保存」）后，才会被写成新 JSON 格式，之前一直保持原样。

**PPT 导出兼容**

`pptx_utils.py` 已同步更新，导出时自动将 JSON 清单转为可读文本：

```
✓ 已完成的条目
· 未完成的条目
```

旧纯文本字段原样输出，不受影响。

**升级操作**

无额外步骤，按 5.2 标准流程执行即可：

```bash
git pull origin main
sudo systemctl restart aipm-backend
cd frontend && npm install && npm run build
sudo systemctl reload nginx
```

#### v0.12.x → 新增项目一本通 + 专项管理

**变更内容**

1. 新增页面「项目一本通」：管理员维护自定义分类与条目（外链/文件上传），普通用户只读、可下载。
2. 新增页面「专项管理」：管理员配置专项列表，每个专项一个详情页（专项目标 / 里程碑计划 / 一句话进展 / 全景图 / 事务表 / 风险问题表 / 阵型）。
3. 左侧导航的"专项管理"为二级菜单，展开后可见每个启用专项；管理员额外多一项"专项配置"用于增删改专项。
4. 新建数据库表：`handbook_categories` / `handbook_items` / `specials` / `special_contents` / `special_tasks` / `special_risks`。

**是否需要修改数据库？不需要手动操作。** SQLAlchemy `create_all` 会在后端启动时自动建表。

**新增文件上传目录**

- 一本通文件：`backend/uploads/handbook/<yyyymm>/<uuid>.<ext>`
- 专项全景图：`backend/uploads/specials/<special_id>/<uuid>.<ext>`

升级时确保后端进程对 `backend/uploads/` 目录有写权限：

```bash
mkdir -p /opt/aipm/app/backend/uploads
chown -R aipm:aipm /opt/aipm/app/backend/uploads
```

**备份建议**

`backend/uploads/` 是用户上传的文件，须与 `app.db` 一起备份：

```bash
# 在 5.3 的备份脚本中追加
tar -czf "$BACKUP_DIR/uploads-$(date +%F).tar.gz" -C /opt/aipm/app/backend uploads
```

**升级操作**

无额外步骤，按 5.2 标准流程执行即可。升级后用 admin 登录，先去「专项管理 → 专项配置」新增一个专项试一下。

#### v0.11.x → 新增操作日志表 + 闲置自动登出

**变更内容**

1. 新增 `operation_logs` 表，记录登录与关键写操作（新增/修改/删除/导入/导出 PPT/运行脚本/修改配置 等），管理员可在前端「操作日志」页面分页查询、过滤。
2. 前端新增闲置自动登出：默认 **15 分钟** 无鼠标/键盘/滚动事件即清空会话并跳转登录页；多 tab 通过 `localStorage` 共享活动时间，任一 tab 在使用其它 tab 都不会被判定为闲置。
3. 任一 tab 主动登出（点「退出登录」、修改密码、收到 401）会通过 `localStorage` 事件广播给其它 tab 同步退出。

**是否需要修改数据库？不需要手动操作。**

`operation_logs` 表会在后端重启时由 SQLAlchemy 的 `create_all` 自动创建，与现有数据无冲突。

**注意事项**

- 操作日志只增不删，若长期运行体量较大，可在 SQLite 中按时间删除老数据：
  ```sql
  -- 例如保留最近 6 个月
  DELETE FROM operation_logs WHERE created_at < datetime('now','-6 months');
  VACUUM;
  ```
- 闲置阈值若需调整，修改 [frontend/src/App.vue](../frontend/src/App.vue) 中的常量 `IDLE_MS`。
- 前端会在「退出登录」时主动调用 `POST /api/auth/logout`，仅用于写一条登出日志，后端无服务端会话状态。

**升级操作**

无额外步骤，按 5.2 标准流程执行即可。

## 6. 常见问题排查

### 后端无法启动

```bash
sudo journalctl -u aipm-backend -n 50
```

常见原因：
- Python 版本不符：确认 `venv/bin/python --version` 为 3.11+
- 端口占用：`sudo ss -tlnp | grep 8000`
- `config.json` 格式错误：用 `python3 -m json.tool config.json` 验证

### 前端页面空白 / 404

```bash
sudo nginx -t                          # 检查 Nginx 配置语法
ls /opt/aipm/app/frontend/dist/        # 确认构建产物存在
sudo tail -f /var/log/nginx/error.log  # 查看 Nginx 错误日志
```

### API 请求 502 Bad Gateway

后端服务未运行或崩溃：

```bash
sudo systemctl status aipm-backend
sudo systemctl restart aipm-backend
```

### 忘记 admin 密码

直接在数据库中重置（需要 Python）：

```bash
cd /opt/aipm/app/backend
source venv/bin/activate
python3 - <<'EOF'
from auth import hash_password
from database import SessionLocal
import models

db = SessionLocal()
user = db.query(models.User).filter(models.User.username == "admin").first()
user.password_hash = hash_password("新密码")
db.commit()
print("密码已重置")
db.close()
EOF
```

### 查看实时日志

```bash
# 后端日志（实时）
sudo journalctl -u aipm-backend -f

# Nginx 访问日志
sudo tail -f /var/log/nginx/access.log
```
